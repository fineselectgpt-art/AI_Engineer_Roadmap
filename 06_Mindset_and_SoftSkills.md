## 6.4 Debugging Mindset

When a model performs poorly:
1. verify data split + leakage
2. verify labels are correct
3. create a dumb baseline
4. run error analysis
5. change one thing at a time

When training breaks:
- check shapes
- check learning rate
- overfit a tiny subset (sanity test)
- log everything
